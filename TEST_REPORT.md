## 🚨 严重问题 (P0 - 必须修复)

### 1. [P0-001] 选域页 - "待补"状态造成用户困惑
**位置**: `src/pages/Path.tsx`
**问题描述**: 当 domain 的 `ready: false` 时，显示"待补/Forthcoming"，用户点击无反应但 cursor 仍为 pointer
**影响**: 用户体验中断，不知道这个选项什么时候可用
**建议修复**:
- 禁用时 cursor 改为 `not-allowed`
- 添加 tooltip 说明"该领域正在开发中"
- 或者用渐进式显示，只显示已就绪的域

### 2. [P0-002] 答题页 - 完成题目数与实际不符
**位置**: `src/pages/Way.tsx` line 96-97
**代码**:
```typescript
const canFinish = answeredCount >= 30;
```
**问题**: 题目总共是 48 道，但 `canFinish` 只要求 30 道，可能导致结果不够准确
**影响**: 匹配结果可能不准确
**建议修复**: 将 30 改为 48，或根据实际需求调整阈值

### 3. [P0-003] 分享卡 - 字体依赖外部系统
**位置**: `src/components/ShareCard.tsx` line 150-160
**问题**: 硬编码使用 `STXingkai`, `KaiTi`, `SimKai` 等中文字体，用户设备可能没有这些字体
**影响**: 分享卡在很多设备上显示为默认字体，破坏设计一致性
**建议修复**:
- 使用 WebFont 加载或 SVG 文字渲染
- 或者提供 fallback 字体列表

## ⚠️ 中等问题 (P1 - 应该修复)

### 4. [P1-001] 顶部导航 - Logo 点击行为不明确
**位置**: `src/components/TopBar.tsx`
**问题**: 点击 Logo 会弹出确认框，但在未答题时也应直接返回
**建议**: 
- 未答题时直接跳转
- 已答题时弹出确认框

### 5. [P1-002] 答题页 - 回溯提示位置尴尬
**位置**: `src/pages/Way.tsx` line 113-116
**问题**: `backtrackHint` 显示在顶部 controls 旁边，与其他信息混在一起
**建议**: 考虑移至答题区下方或作为单独状态提示

### 6. [P1-003] 键盘导航 - 提示文字过小
**位置**: `src/index.css` line 568
**代码**: `.cp-way-hint { font-size: 0.6rem; }` (移动端)
**问题**: 在超小屏幕上字体仅 0.6rem (约 9.6px)，几乎不可读
**建议**: 移动端最小字体限制为 12px

### 7. [P1-004] 主题切换 - 没有过渡动画
**位置**: `src/store.ts`
**问题**: 主题切换是瞬时的，没有过渡动画
**建议**: 添加 CSS transition 让颜色变化更平滑

### 8. [P1-005] 人物详情弹窗 - 移动端布局问题
**位置**: `src/components/FigureDetail.tsx`
**问题**: 移动端网格改为单列后，肖像与文字之间的间距需要调整
**建议**: 优化移动端 padding 和间距

### 9. [P1-006] 英文翻译 - 部分文案生硬
**位置**: `src/i18n/en.ts`
**问题**: 
- `ConfidenceHint`: "confidence ~85% · Early answers, compass still forming" 文法可优化
- `backtrackHint`: "Some questions ahead remain unanswered" 语气不够优雅
**建议**: 优化英文语调，与中文古风相配

### 10. [P1-007] 映照页 - 十二维卡片过密
**位置**: `src/pages/Reflection.tsx` 和 `src/index.css`
**问题**: 桌面端 12 个特质卡片排列紧密，每个卡片内部信息过多
**建议**: 
- 增加桌面端网格列数
- 或者折叠部分信息为 expandable

### 11. [P1-008] 置信度提示 - 位置不够醒目
**位置**: `src/pages/Reflection.tsx` line 170-182
**问题**: 当置信度 < 70% 时提示信息较弱
**建议**: 考虑在顶部醒目位置或雷达图旁边显示置信度

### 12. [P1-009] Toast 提示 - 消失太快
**位置**: `src/pages/Reflection.tsx` line 255
**代码**: `toastTimer.current = setTimeout(() => setToast(null), 1500);`
**问题**: 1.5 秒对于较长的提示可能不够
**建议**: 根据提示长度动态调整时间

## ♿ 无障碍问题 (A11y)

### 20. [A11Y-001] 进度条 - 颜色对比度
**位置**: `src/index.css` line 430-435
**问题**: 进度条渐变色可能不符合 WCAG AA
**建议**: 使用足够对比的颜色

### 21. [A11Y-002] Skip Link - 移动端不可见
**位置**: `src/index.css` line 650-665
**问题**: Skip Link 在移动端可能遮挡主要内容
**建议**: 增加 z-index 或调整定位

### 22. [A11Y-003] 聚焦指示器 - 在暗色模式下不够醒目
**位置**: `src/index.css` line 686-692
**问题**: 暗色模式下 `outline-color: var(--cinnabar)` 可能不够醒目
**建议**: 增加 outline 宽度或添加 box-shadow

### 23. [A11Y-004] ARIA Live Region - 使用不当
**位置**: 多个组件
**问题**: 部分使用 `aria-live="polite"` 但内容是静态的
**建议**: 只在动态内容变化时使用 aria-live

### 24. [A11Y-005] 肖像替代文本
**位置**: `src/components/Portrait.tsx`
**问题**: 肖像图片可能缺少有意义的 alt 文本
**建议**: 添加 `aria-label` 包含人物名字

### 25. [A11Y-006] 触控目标尺寸
**位置**: `src/index.css` line 696-700
**问题**: 虽然有 `min-height: 44px` 但按钮实际可能更小
**建议**: 确保所有可交互元素满足 44x44px 触控目标

## 🚀 性能问题 (Performance)

### 37. [PERF-001] 肖像图片未优化
**问题**: 150 张肖像 SVG 可能较大
**建议**: 
- 考虑 SVG 压缩
- 使用 `<picture>` 标签提供不同尺寸

### 38. [PERF-002] GSAP 动画可能造成重排
**位置**: 多个组件
**问题**: 使用 GSAP 修改某些属性可能触发重排
**建议**: 优先使用 transform 和 opacity

### 39. [PERF-003] 未使用 React.memo
**位置**: 多个组件
**问题**: 大型组件如 Reflection 和 Way 可能在每次状态变化时重新渲染
**建议**: 使用 React.memo 或 useMemo 优化
