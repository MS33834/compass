# 指南 · AI 交付团队专家角色定义

> 本文档定义"指南（Compass）"项目 AI 交付团队的角色分工、职责边界、交付物与考核标准。
> 所有 AI 专家以子代理（subagent）形式并行调度，由总调度（主对话）统一编排与验收。
> 版本：v1.0 · 2026-06-24

---

## 一、团队组成与协作模型

```
                    ┌─────────────────┐
                    │   总调度 (主对话)  │  ← 用户唯一接口
                    │  编排 / 验收 / 同步 │
                    └────────┬────────┘
                             │
        ┌────────┬───────────┼───────────┬────────┐
        ▼        ▼           ▼           ▼        ▼
   ┌────────┐┌────────┐┌─────────┐┌────────┐┌────────┐
   │ 项目经理 ││前端架构师││QA 测试工程师││DevOps ││内容国际化│
   │  PM    ││ Arch   ││   QA    ││ 工程师  ││  专家   │
   └────────┘└────────┘└─────────┘└────────┘└────────┘
        │        │           │           │        │
        ▼        ▼           ▼           ▼        ▼
   ┌────────┐┌────────┐┌─────────┐┌────────┐┌────────┐
   │产品经理 ││视觉设计││技术文档工程师││ 小程序  ││  数据   │
   │  PD    ││  VD    ││   TW    ││ 工程师  ││ 工程师  │
   └────────┘└────────┘└─────────┘└────────┘└────────┘
```

**协作原则：**
1. **并行优先**：无依赖关系的任务必须并行调度，缩短交付周期。
2. **单一职责**：每个专家只对自己的交付物负责，跨域问题由 PM 协调。
3. **验收前置**：每个专家开工前明确"完成标准（DoD）"，由总调度验收。
4. **双仓同步**：每完成一个阶段，由总调度统一推送到 GitHub + GitCode。

## 四、验收与同步机制

1. **每波结束**：总调度运行 `npm run test:all` + `format:check` + `build` 全绿。
2. **双仓同步**：总调度执行 `git push origin main && git push gitcode main`。
3. **令牌安全**：访问令牌仅存在于本地 remote URL，禁止写入任何仓库文件。
4. **阶段报告**：每波结束输出简要验收报告，更新本文件"推进计划"表状态。
